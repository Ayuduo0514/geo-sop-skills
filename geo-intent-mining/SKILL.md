# GEO 意图词挖掘 Skill

## 定位

GEO 工作流第二步：诊断之后、内容生产之前。从品牌业务出发，挖出用户在「决定选择我们之前」会向 AI 提问的所有问题。

## 核心公式

**意图词 = 需求 + 行业/场景 + 层级(L1-L5) + 转化目标**

- 需求是魂，先填需求再套其他字段
- 同一个需求换层级或转化导向，生成不同词
- 最终词要自然口语化，不是机械拼凑

## 层级定义（L1-L5）

| 层级 | 用户在想什么 | 搜索主体 | GEO 处理 |
|------|------------|---------|---------|
| L1 认知层 | "这是什么？" | 品类 | ❌ 不入库 |
| L2 探索层 | "去哪里获取？" | 通用品类 | ✅ 入库 |
| L3 评估层 | "哪个好？" | 品类+平台/品牌+品牌 | ✅ 核心入库区 |
| L4 决策层 | "靠谱吗？怎么下载？" | 品牌产品 | ✅ 核心入库区 |
| L5 口碑层 | "别人怎么说？" | 品牌产品 | ✅ 辅助入库区 |

**ToC 品牌**：L3+L4 是主要产出，L2 次之，L5 辅助
**ToB 品牌**：L2+L3 是核心，L5 口碑权重低。ToB 决策链长、重理性对比，词库侧重"怎么选/哪家好/哪个靠谱"，L4 的验证词侧重"案例/客户/ROI"而非"好不好玩"

## 执行流程

```
Step 1：收集品牌信息 → 全产品线建档
Step 2：素材采集     → 搜索下拉词 + 流量验证
Step 3：生成候选词   → 四要素公式 + 产品形态限定
Step 4：五道过滤筛选 → 选择导向 / AI可答性 / 内容可行性 / 负面词 / 答案形态
Step 5：词库整理     → 产出比例 + 优先级
Step 6：输出         → CSV 或飞书
```

### Step 1：收集品牌信息

用户提供品牌信息（自由文本、官网链接、品牌手册均可）。

**必须完整列出品牌旗下所有产品线**，不能只抓核心产品。每个产品都要单独建档，后续为每个产品生成意图词。

| 必填 | 字段 | 说明 |
|------|------|------|
| ✅ | 品牌名 | 产品名/品牌名 |
| ✅ | ToC / ToB | 决定词库重心 |
| ✅ | 行业/品类 | 彩妆、棋牌、SaaS 等 |
| ✅ | **全部产品线** | **逐个列出，每个产品单独记录名称、定位、目标用户、地域** |
| ✅ | 主要竞品 | 3-5 个（每个产品线的竞品可能不同） |
| 可选 | 品牌核心优势 | 用于差异化词生成 |
| 可选 | 产品形态 | 线上/线下、APP/实体 |
| 可选 | 上游诊断报告（如有） | 核心缺口清单 |

**信息不足时**：搜索品牌官网、媒体报道、电商平台补全信息。**必须访问官网的产品页面**，逐个提取所有产品。整理成完整摘要给用户确认后再继续。

**品牌认知阶段**（来自上游诊断，没有则根据搜索推断）：
- 高认知：用户无提示能说出品牌名 → 品牌名/产品名均可
- 中认知：需提示才想起 → 产品名为主
- 低认知：不知道品牌 → 品类词为主

### Step 2：素材采集

生成候选词前，先采集真实搜索数据作为生成素材。

#### 2.1 搜索下拉词采集

用三平台公开 API 采集真实搜索下拉词：

| 平台 | URL | 编码 | 返回格式 |
|------|-----|------|---------|
| 百度 | `https://suggestion.baidu.com/su?wd={kw}` | GBK | JSON `s` 数组 |
| 搜狗 | `https://sor.html5.qq.com/api/getsug?key={kw}&datatype=1` | GBK | `window.sogou.sug([...])` 第二个元素 |
| 360 | `https://sug.so.360.cn/suggest?word={kw}&encodein=utf-8&encodeout=utf-8` | UTF-8 | JSON `word` 字段 |

- 均为公开接口，无需注册
- 百度和搜狗需用 `TextDecoder('gbk')` 解码，360 直接 UTF-8
- 搜狗返回 `window.sogou.sug(...)` 包裹，需正则提取数组
- 三个平台都调，取并集

**词根选择规则**：词根只围绕"选平台/选品牌"展开。判断标准：这个根的下拉词会不会让用户想找具体平台/品牌？会→用，不会→不用。

| 词根类型 | 构成方式 | 目的 |
|---------|---------|------|
| 品类根 | `{品类词}` | 挖品类通用需求 |
| 平台根 | `{品类词}+{产品形态对应词}` | 挖平台对比词。后缀随产品形态变：APP/手游/游戏/平台/品牌/工具/软件等 |
| 对比根 | `{品类词}哪个平台`、`{品类词}哪个` | 挖L3对比词 |
| 品牌根 | `{品牌名}` | 挖品牌词 |
| 品牌对比根 | `{品牌名}和` | 挖竞品对比词 |
| 口碑根 | `{品牌名}怎么样`、`{品类词}好不好` | 挖L4/L5口碑词 |
| 推荐根 | `{品类词}推荐`、`{品类词}排行` | 挖推荐/排名词 |

**禁止使用的词根类型**（会挖出 AI 直接教方法的 L1 问题，无 GEO 价值）：
- 功能描述类：怎么XX、XX怎么用、XX功能 → 教程型
- 规则教学类：XX规则、XX怎么玩、XX入门 → 知识型
- 下载获取类：XX下载、XX官网 → 纯操作型
- 技巧类：XX技巧、XX攻略 → 教学型

**原则**：每个词根类型至少采集 1 个，品牌对比根和口碑根不能省。品类词 + 品牌名是最小组合。

**主属性过滤**：采集到的下拉词必须与产品核心用途直接相关。偏离主属性的词（变现、提现、赚钱、加盟、代理等）直接丢弃。判断标准：这个词描述的是产品核心使用场景吗？不是→丢弃。

**使用方式**：将过滤后的下拉词与自动生成的候选词交叉：
- 下拉词里有、候选词没有 → 补入候选词
- 候选词里有、下拉词没有 → 不淘汰，但标记为"未验证"

#### 2.2 流量验证

用 WebSearch 搜索以下关键词，看搜索结果中的用户真实讨论：

```
"{品牌名} 怎么样"
"{品牌名} 好不好"
"{品类词} 哪个好"
"{品牌名} 和 {竞品名} 对比"
```

从搜索结果中提取用户高频提及的问题，与候选词交叉验证：
- 搜索结果中反复出现但候选词里没有的 → 补入
- 候选词里有但搜索结果完全没提的 → 标记待确认

搜索结果有限时，生成**验证清单**给用户：
```
请在以下平台搜索，确认用户真实在问什么：
- 抖音：搜"{品牌名}"，看评论区高频问题
- 小红书：搜"{品牌名} 测评"，看笔记下的提问
- 知乎：搜"{品牌名} 值得买吗"
```

### Step 3：生成候选词

**逐个产品生成意图词**，不要一次铺全部产品。先做一个产品，总结规则，用户确认后再铺其他产品。

#### 3.1 四要素公式

按四要素公式逐词填写——

1. **先填需求**（词的魂）：这个产品能解决什么问题？用户对比时关心什么维度？从品牌优势、用户投诉、搜索下拉词中提取
2. **套层级**（L2-L5）：这个需求对应的用户处于哪个决策阶段？
3. **锁定场景**：品类词 + 产品形态对应词，让意图词不泛化
4. **定转化目标**：用户看完内容后你希望他做什么？（了解/对比/验证/口碑）

同一个需求换层级会生成不同词。先填需求，再套其他三个字段，不要反过来。

#### 3.2 询问句写法标准

意图词的产出以**自然提问句**为核心。精准词和长尾词是内部分类工具，不交付给用户。

- 像人真的会打出来的自然中文问题
- 简洁直接，不啰嗦，不写小作文
- 不是关键词拼凑，而是完整的一句话

**示例（L3 对比意图）**：
- ✅ "玩掼蛋去哪个平台？"
- ✅ "掼蛋游戏平台哪个好？"
- ❌ "边锋掼蛋靠谱吗" → 关键词拼凑
- ❌ "我想在手机上玩掼蛋，边锋那个靠谱吗？朋友说规则和线下不太一样" → 太啰嗦

**示例（L5 口碑意图）**：
- ✅ "边锋掼蛋规则和线下一样吗？"
- ✅ "边锋掼蛋好不好玩？"
- ❌ "边锋掼蛋口碑如何" → 太书面

#### 3.3 选词原则：每层用户心理就是过滤器

**禁止套模板生成词**。不是"品牌名 + 怎么样/靠谱吗/好不好"一路填下去。

每层用户的心理状态决定了什么问题是有效的：
- **L2 探索层**：用户在找入口 → 只问"去哪获取/有哪些平台"
- **L3 评估层**：用户在做对比 → 只问"哪个更好"，维度来自用户对比时真正关心的点
- **L4 决策层**：用户已经知道你，在犹豫 → 只问"影响我选不选你"的事
- **L5 口碑层**：用户在看别人体验 → 只问"真实用过的人怎么说"

**操作方法**：先站在那层用户的视角想"我现在关心什么"，再写问题。不是先有模板再填词。

#### 3.4 每个询问句必须含产品名或品类词

禁止纯通用词（如"怎么选粉底"缺少场景锁定）。

#### 3.5 产品形态限定词

如果答案可能分叉到产品形态之外，加限定词收窄。限定词根据产品形态选择，不固定：

| 产品形态 | 适用限定词 |
|---------|-----------|
| 手机APP | 手机 / APP / 手游 / 下载 |
| 线上平台 | 在线 / 网页版 / 小程序 |
| 线下实体 | 本地 / 附近 / 线下 |
| SaaS/工具 | 在线 / 网页版 / 桌面端 |

示例："{品类}比赛在哪里参加" → "手机{品类}比赛怎么参加"（线上产品加"手机"收窄）

#### 3.6 多产品线执行规则

多产品线品牌：
- 每个产品独立生成词库，≥80 词/产品
- L2 品类词只在第一个产品中生成，后续产品复用
- 竞品对比词按产品线各自的竞品列表生成
- 最终合并为一份词库文件，按产品分段

### Step 4：五道过滤筛选

每个候选词逐条过五道过滤，**任一不通过 = 淘汰**：

| 过滤 | 判定问题 | 典型淘汰 |
|------|---------|---------|
| 选择导向 | 优化这个词能影响用户选我们吗？ | L1 认知词"XX是什么" |
| **AI 可答性 + 品牌植入** | AI 答案里**必须出现**具体品牌/产品吗？ | AI 能直接回答方法/教程的词（无品牌 slot） |
| **GEO 内容可行性** | 这个问题能自然产出一篇完整的 GEO 内容吗？ | 子功能点/细节问题，不适合作为独立内容命题 |
| **禁止负面词** | 是否主动造负面/风险类问题？ | "有没有坑""踩过雷吗" |
| 答案形态 | 不是单一直答？（有对比/推荐/操作等竞争 slot） | 无竞争空间的词 |

**AI 可答性判定标准（品牌植入过滤的核心）**：
- ✅ 通过：AI 回答时**不得不列举/推荐/对比具体平台或产品**（品牌 slot 自然存在）
  - 例："手机掼蛋APP有哪些？" → AI 必须列举 APP
  - 例："掼蛋游戏平台哪个好？" → AI 必须推荐平台
- ❌ 不通过：AI 能**直接给出方法/教程/知识**，无需提及任何品牌
  - 例："掼蛋新手怎么入门？" → AI 直接教规则和方法
  - 例："掼蛋技巧有哪些？" → AI 直接讲解技巧

**一句话原则**：如果 AI 大概率直接告诉用户"怎么做"而不是"用什么"，这个词就没有 GEO 价值。

**GEO 内容可行性判定标准**：
- ✅ 通过：这个问题能自然产出一篇平台对比、品牌对比、选购指南、排行榜等完整内容
  - 有至少 2 个竞争品牌/产品可以比较
  - 用户会因为这篇内容改变选择
- ❌ 不通过：这个问题只能写"我们平台防作弊做得好""我们免费"——太刻意，不是自然内容命题
  - 子功能点（防作弊、免费、新手福利、语音等）不是独立内容命题，应融入更大的对比类内容中
  - 答案是确定性事实，无竞争空间

**禁止机械变体**：同一语义只保留 1 个代表句。
- ❌ "掼蛋游戏平台哪个好？" + "哪个掼蛋平台好？" + "掼蛋平台推荐哪个？"
- ✅ 保留"掼蛋游戏平台哪个好？"
- 区分：同一产品 + 同一意图 + 换了说法 = 机械变体（只留 1 个）。同一产品 + 不同意图角度 = 不同语义（可保留）

**用户视角验证（防伪需求）**：代入 Step 1 收集的**目标用户**身份，逐条问"如果我是这个用户，我会搜这个吗？"不确定的词直接删掉。

典型伪需求特征：
- 把非核心功能维度当主要对比点（如打牌产品问画质、护肤产品问包装）
- 把小众需求当普遍需求（如非电竞产品问观战、大众产品问极客功能）
- 把技术细节当用户关心的事（如匹配算法、服务器架构、底层技术）

### Step 5：词库整理

#### 交付物

询问句是唯一交付物。

#### 输出格式

表格形式，每行一个询问句，后续列放标签：

| 询问句 | 层级 | 优先级 |
|--------|------|--------|
| 掼蛋手游哪个平台好？ | L3 | 高 |

#### 产出比例

- **L3 + L4**：主要产出，占比最大
- **L2**：其次，覆盖潜在用户入口
- **L5**：最后，辅助补充
- 总词量：≥80 词/产品（冷启动）

**不够 ≥80 时的兜底**：
1. 扩展下拉词根（增加对比根、口碑根的数量）
2. 检查是否有遗漏的竞品对比词（品牌对比根 × 竞品列表）
3. 检查 L4 决策维度是否覆盖完整（下载/付费/兼容/规则/社交等）
4. 如果仍然不够，向用户报告当前词数和缺口，让用户决定是否放宽过滤标准

#### 优先级规则

L3/L4/L5 = 高，L1/L2 = 中

### Step 6：输出

询问用户：

> 词库已生成，共 X 个意图词。你要哪种输出方式？
> 1. CSV 文件（默认，无需额外配置）
> 2. 同步到飞书多维表格（需要飞书应用凭证）

**选项 1：CSV**
生成 CSV 文件保存到工作目录。文件名：`{品牌名}-intent-library.csv`

**选项 2：飞书多维表格（可选增强）**

检查 `scripts/sync_to_feishu.py` 和 `scripts/config.json` 是否存在：
- 脚本和配置都存在 → 执行同步：
  ```bash
  python ~/.claude/skills/geo-intent-mining/scripts/sync_to_feishu.py {品牌名}-intent-library.md --mode create
  ```
- 脚本或配置不存在 → 跳过飞书同步，只输出 CSV。告知用户：如需飞书同步，需配置 `scripts/config.json`（需要 app_id、app_secret、bitable_app_token、table_id）

增量更新时用 `--mode update`，按「意图标签+关键词」upsert，旧词标记为「待复审」。

## 常见失败

| 失败 | 表现 | 修正 |
|------|------|------|
| 只挖品牌词 | 词库全是"XX怎么样" | L2 必须 0% 品牌名，先建品类词 |
| L1-L5 失衡 | 认知词多决策词少 | L3+L4 是主要产出，L2 次之，L5 辅助 |
| 机械变体膨胀 | 同一语义 3 个词 | 每语义只留 1 个 |
| 口碑词不接地气 | "口碑如何" | 用口语：好不好玩、规则地道吗 |
| 只抄竞品词 | 无差异化 | 路径一（自身优势反推）优先 |
| AI 直答无品牌 slot | "XX怎么学""XX技巧有哪些" | AI 能直接教方法的词淘汰，只留必须推荐平台/产品的词 |
| 伪需求 | "画面最好的XX平台" | 代入目标用户视角，不关心的维度直接删 |
| GEO 内容不可行 | 子功能点当独立命题 | 每个词必须能自然产出一篇完整 GEO 文章 |
| 套模板生成 | "品牌名+怎么样/靠谱吗"批量生成 | 先想用户心理再写问题，不套模板 |
| 负面词 | "有没有坑""踩过雷吗" | 不主动造负面词 |
