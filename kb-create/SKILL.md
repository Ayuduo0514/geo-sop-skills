# kb-create — 知识库页面强制模板

## 定位

知识库页面的**源头门禁**。新建页面不走模板 = 后续 kb-qa 判 P0 阻断。

**执行时机**：Claude Code 创建任何 知识库/ 下的 .md 文件之前。先跑模板，再写内容。

---

## 页面类型判断

根据用户意图或文件路径判断类型：

| 用户说法 | 页面类型 | 存放目录 |
|---------|---------|---------|
| "新建概念" / "概念页" | 概念 | `知识库/概念/` |
| "新建实体" / "实体页" / "品牌页" | 实体 | `知识库/实体/` |
| "新建来源摘要" / "摘要" / "ingest 资料" | 来源摘要 | `知识库/来源摘要/` |
| "新建 SOP" / "标准操作" / "流程" | SOP | `知识库/综合/` |

**不确定时问用户**："这是概念页、实体页、来源摘要还是 SOP？"

## 执行纪律

1. **绝不空着 frontmatter 放行**。`可信度来源:` 至少填一个，不确定填 `[TODO: 补充来源]`。
2. **文件名规范**：kebab-case，中文文件名允许但优先英文（便于 wikilink）。
3. **创建后追加日志**：在 `知识库/日志.md` 写一行：`## [YYYY-MM-DD] create | 新建 {页面类型}/{文件名}`。
4. **概念页/实体页必须互链**：新建概念页时，检查是否有相关实体页需要更新；反之亦然。

## 常见错误

| 错误 | 后果 | 修复 |
|------|------|------|
| 跳过模板直接写内容 | kb-qa Q7 P0 阻断 | 删除重来，先补 frontmatter |
| `最后更新:` 填创建日期以外的值 | Q16 生命周期检查误报 | 必须为实际创建日期 |
| `可信度来源:` 为空 | 知识库可信度无法追溯 | 至少填 `[TODO: 补充来源]` |
| SOP 缺少五节骨架 | kb-qa Q10 P1 严重 | 按模板补齐 |
| 来源摘要没有 `---` 分隔 | compiled truth 和 timeline 混在一起 | 插入 `---` 分隔线 |
