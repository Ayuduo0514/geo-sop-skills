# kb-check pre-write — 写入前拦截器

## 定位

**事前门禁，不是事后审计。** 在内容落盘前最后一道防线，把结构性问题挡在库外。

**执行时机**：Write/Edit 工具调用前。**必须先执行 pre-write，通过后才可以写入。**

**与 kb-check 的关系**：
- `kb-save` = 写入前快速拦截（L1  only，30 秒）
- `kb-check` = 改动完成后全量审计（17 项，完整报告）

---

## 执行纪律

1. **强制顺序**：先 pre-write → 通过后 → 再 Write/Edit。顺序不可颠倒。
2. **P0 阻断**：Q1/Q3/Q11 任一项未通过 = **绝对禁止写入**，没有例外。
3. **自动修复优先**：Q7/Q12/Q13 发现问题先自动修复，修复后重扫通过才允许写入。
4. **超时回退**：如果 60 秒内无法完成扫描（如文件系统异常），**禁止写入**并报告异常。
5. **编码合规**：如果使用 PowerShell `Set-Content`、Node.js `fs.writeFileSync` 等外部工具写中文文件，必须显式指定 UTF-8 编码。PowerShell 未加 `-Encoding UTF8` 或 Node.js 未加 `'utf8'` = **禁止写入**（`BAD_ENCODING`）。

## L1 扫描项（7 项）

### Q1. 目录结构合规（P0 — 阻断级）

**检查**：写入目标路径是否在正确的子目录？

| 文件类型 | 正确目录 | 错误示例 |
|---------|---------|---------|
| 概念页 | `知识库/概念/` | `知识库/综合/概念_xxx.md` |
| 实体页 | `知识库/实体/` | `知识库/概念/实体_xxx.md` |
| 来源摘要 | `知识库/来源摘要/` | `知识库/综合/来源_xxx.md` |
| SOP | `知识库/综合/` | `知识库/概念/SOP_xxx.md` |
| AI 操作记录 | `知识库/AI操作记录/` | `知识库/综合/plan_xxx.md` |
| 索引/日志 | `知识库/` 根目录 | 放进子目录 |

**禁止写入原始资料/**：任何写入 `原始资料/` 目录的操作直接拦截（`RO_VIOLATION`）。

---

### Q3. 死链检测（P0 — 阻断级）

**检查**：待写入内容中的 `[[xxx]]` 链接指向的文件是否存在？

**执行**：
1. 提取待写入内容中的所有 `[[...]]` wikilink
2. 对每个 wikilink，检查目标文件是否存在于知识库目录
3. 不存在的 = 死链

**自动修复**：
- 概念页死链 → 创建 `概念/XXX.md` stub（最小 frontmatter + 定义占位）
- 实体页死链 → 创建 `实体/XXX.md` stub（最小 frontmatter + 基本信息占位）
- 来源摘要/SOP 死链 → **不自动修复**，报告错误（`DEAD_LINK_CRITICAL`）

---

### Q7. Frontmatter 完整性（P1 — 尝试自动修复）

**检查**：待写入内容是否有正确的 frontmatter？

| 文件类型 | 必填字段 | 缺失时的自动修复 |
|---------|---------|----------------|
| 概念页 | `type: 概念`、`可信度来源:`、`最后更新:` | 补全缺失字段 |
| 实体页 | `type: 实体`、`实体类型:`、`最后更新:` | 补全缺失字段 |
| 来源摘要 | `type: 来源摘要`、`原始资料:`、`最后更新:` | 补 `type` 和 `最后更新`；`原始资料` 不自动补 |
| SOP | `类型:`、`编号:`、`名称:`、`最后更新:` | 补全缺失字段 |

---

### Q11. 原始资料只读（P0 — 阻断级）

**检查**：写入目标路径是否包含 `原始资料/`？

**规则**：
- 原始资料目录 = **只读**，任何 Edit/Write 操作直接拦截
- 唯一允许的例外：ingest 流程中追加 `## 引用关系` 回链（且不得修改正文）

---

### Q12. Wikilink 格式（P1 — 尝试自动修复）

**检查**：内部链接是否使用 `[[xxx]]` 格式？

**拦截**：发现 `[文字](路径.md)` 格式的内部链接。

**自动修复**：转换为 `[[路径|文字]]` 或 `[[路径]]`。外部 URL 保持不变。

---

### Q13. TODO 占位（P1 — 尝试自动修复）

**检查**：待写入内容中是否有 `[TODO: ...]`？

**规则**：
- 正文中有未处理的 `[TODO: ...]` → 拦截并报告
- `AI操作记录/` 中的 TODO → 允许（属于进行中任务）

---

### Q16. 最后更新日期（P1 — 尝试自动修复）

**检查**：frontmatter 中的 `最后更新:` 是否为今天？

**自动修复**：如果缺失或日期不是今天，更新为当前日期。

## 错误码速查

| 错误码 | 级别 | 含义 | 自动修复 |
|--------|------|------|---------|
| `WRONG_DIR` | P0 | 文件类型与目录不匹配 | ❌ 需人工移动 |
| `RO_VIOLATION` | P0 | 试图写入原始资料/ | ❌ 禁止 |
| `DEAD_LINK` | P0 | 死链（来源摘要/SOP） | ❌ 需人工确认 |
| `DEAD_LINK_STUB` | P0→P1 | 死链（概念/实体）自动修复后 | ✅ 已修复 |
| `MISSING_FM` | P1 | frontmatter 缺失字段 | ✅ 自动补全 |
| `BAD_LINK_FMT` | P1 | 内部链接用 markdown 格式 | ✅ 自动转 wikilink |
| `TODO_LEFT` | P1 | 正文有未处理 TODO | ❌ 需人工处理 |
| `STALE_DATE` | P1 | 最后更新日期不是今天 | ✅ 自动更新 |
| `BAD_ENCODING` | P0 | 外部工具写中文文件未指定 UTF-8 | ❌ 需修正命令 |

## 执行命令模板

**死链扫描（Bash）**：
```bash
# 提取内容中所有 wikilink
echo '$CONTENT' | grep -oP '\[\[\K[^\]]+' | sort -u

# 检查每个链接对应文件是否存在
for link in $LINKS; do
  path="D:/Projects/geo-knowledge-base/GEO/GEO理论/知识库/${link}.md"
  if [ ! -f "$path" ]; then
    echo "DEAD_LINK: $link"
  fi
done
```

**Wikilink 格式检查（PowerShell）**：
```powershell
$pattern = '\[([^\]]+)\]\(([^)]+\.md)\)'
$matches = [regex]::Matches($content, $pattern)
if ($matches.Count -gt 0) { Write-Output "BAD_LINK_FMT: $($matches.Count) 处" }
```
