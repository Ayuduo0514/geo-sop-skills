## 1. 四要素公式（挖词总入口）

**公式**：**意图词 = 需求维度 + 行业属性 + 层级标签 + 转化导向**

| 字段 | 说明 | 填法 |
|------|------|------|
| **需求维度** | 用户的核心痛点或诉求 | 从用户投诉/评论/咨询里找抱怨和提问 |
| **行业属性** | 用户所属行业或应用场景 | 品类词 → 场景词 → 人群词，越具体越好 |
| **层级标签** | 用户处于决策旅程的哪个阶段（L1-L5） | 先保证 L3（评估层）和 L5（口碑层）覆盖 |
| **转化导向** | 期望用户看完内容后做什么 | 了解原因 / 了解方法 / 对比方案 / 验证品牌 / 口碑分享 |

**使用原则**：
- 先填需求维度（这是整个词的"魂"），再套其他三个字段
- 同一个需求维度可以生成多个词：换层级、换转化导向，都会生成不同的词
- 公式是方向工具，不是机械拼凑——最终的词要自然口语化

**示例（歌剧魅影 L1）**：
> 需求维度（底妆卡粉）+ 行业属性（彩妆）+ 层级标签（L1 认知层）+ 转化导向（了解原因）
> → 意图词：**底妆上脸半小时就卡粉的原因**
> → 询问句：**底妆卡粉到底是皮肤原因还是产品原因？**

**示例（歌剧魅影 L3）**：
> 需求维度（选一款持妆粉底液）+ 行业属性（彩妆）+ 层级标签（L3 评估层）+ 转化导向（对比选购）
> → 意图词：**2026 年干皮持妆粉底哪个品牌最值入手**
> → 询问句：**歌剧魅影和 XX 粉底哪个干皮更友好？**

## 2. L1-L5 决策旅程（层级标签详解）

| 层级 | 名称 | 用户在想什么 | 词的特征 | 常见分层错误 |
|------|------|-----------|---------|------------|
| **L1** | 认知层 | 感到困扰，但不知道问题叫什么 | 描述现象、问"为什么"，无品牌名 | L1 词里出现品牌名 = 分层错 |
| **L2** | 探索层 | 知道有问题，开始找解决思路 | 出现解决方案类词，但不带品牌 | — |
| **L3** | 评估层 | 了解了方案，开始比较具体选项 | 出现"对比/哪个更好/怎么选" | — |
| **L4** | 决策层 | 基本定了，做最后确认 | 出现品牌名 + 验证类词 | — |
| **L5** | 口碑层 | 使用后，分享或查同类经历 | 出现"真实/测评/踩雷/推荐" | — |

**ToC vs ToB 重心差异**：
- **ToC 品牌**：L1 + L3 + L5 是三个关键层级；L5 口碑词尤其重要
- **ToB 品牌**：L2-L3 是核心，企业采购决策链长、重理性对比，L5 权重相对低
- **中小品牌轻量化**：每层 3-5 个意图标签 × 5-10 个关键词，先保证 L3 和 L5 覆盖

**各层示例（歌剧魅影）**：

**L1 认知层**：
- 精准词：底妆卡粉 / 下午脱妆 / 粉底上脸起皮
- 询问句：底妆卡粉是皮肤原因还是产品原因？为什么换了粉底液还是会卡粉？

**L2 探索层**：
- 精准词：干皮底妆选择 / 持妆粉底怎么用
- 询问句：干皮化底妆容易起皮应该怎么选产品？

**L3 评估层**：
- 精准词：歌剧魅影 vs 花西子粉底 / 中端彩妆品牌对比
- 询问句：歌剧魅影粉底和国货其他品牌比有什么优势？

**L4 决策层**：
- 精准词：歌剧魅影粉底液适合干皮吗 / 歌剧魅影哪款粉底遮瑕力最好
- 询问句：歌剧魅影粉底液 301 号适合黄皮吗？

**L5 口碑层**：
- 精准词：歌剧魅影粉底液测评 / 歌剧魅影是否踩雷
- 询问句：歌剧魅影粉底液到底好不好用？

## 3. 通用五场景骨架

任何新行业，先用这五类场景做 checklist，逐类填充典型问题：

| 场景类型 | 用户在问什么 | 金融行业示例 | 彩妆行业示例 |
|---------|-----------|------------|------------|
| **认知启蒙型** | "XX 是什么"/"XX 有什么用" | "什么是 ETF？" | "底妆卡粉是什么原因？" |
| **对比选购型** | "XX 和 YY 哪个好"/"怎么选 XX" | "货币基金和国债逆回购哪个好？" | "歌剧魅影和花西子粉底哪个干皮更友好？" |
| **操作指导型** | "XX 怎么用"/"XX 怎么操作" | "基金定投具体怎么操作？" | "怎么让底妆更持妆不卡粉？" |
| **品牌评价型** | "XX 品牌怎么样"/"XX 靠谱吗" | "招行理财靠谱吗？" | "歌剧魅影是智商税吗？" |
| **风险规避型** | "XX 有没有风险"/"XX 安全吗" | "信托有哪些风险？" | "彩妆产品里哪些成分容易过敏？" |

**使用方法**：针对你的行业，逐一填写每种场景类型的典型提问，每个场景至少 3-5 个询问句。

**注意**："风险规避型"在不同行业内容差异最大——金融关心资金风险，彩妆关心成分安全，教育关心课程效果。必须按行业调整。

## 4. 三路径挖词方法

### 路径一：从自身优势反推王炸需求

**适用**：品牌有明确竞争优势，需要把优势转化成用户能搜到的词。

**操作公式**：品牌核心优势 → 反推能解决的核心需求 → 形成"王炸问题清单"

**示例（歌剧魅影）**：
> 品牌优势 = "彩妆发色饱和、质地偏哑光，适合复古妆感"
> → 王炸问题：
> - "国货哑光口红哪个品牌发色最准？"
> - "想要复古妆感用什么彩妆品牌？"
> - "歌剧魅影口红和 Mac 哑光唇膏哪个更显白？"

### 路径二：从真实流量验证问题

**适用**：不确定用户真实在搜什么，需要数据验证。

**操作**：找抖音上品牌或同品类产品的高流量视频，看评论区里真实的问题和吐槽。

**关键约束**：不能凭感觉猜需求。你觉得用户关心"成分安全"，但评论区全是"色号显不显白"——那"成分安全"词建了也没用。

### 路径三：人群 × 场景交叉拆精准需求

**适用**：刚接手一个新品牌，需要系统建立词库骨架。覆盖最全的方法。

**操作公式**：**前 3 大目标人群 × 每个人群前 10 个具体场景 = 核心意图词矩阵**

**示例（歌剧魅影）**：

| 目标人群 | 场景 | 生成的意图词/询问句 |
|---------|------|-----------------|
| 彩妆新手 | 第一次买彩妆 | "彩妆新手第一次应该买哪些入门产品？" |
| 通勤上班族 | 需要全天持妆不补妆 | "上班族化什么底妆全天不花？" |
| 敏感肌用户 | 担心彩妆过敏 | "敏感肌用歌剧魅影彩妆会过敏吗？" |

## 5. 词库三层结构与阈值

### 三层结构

```
意图标签（一个阶段的核心主题）
  └─ 关键词组
        ├─ 精准词（核心短词，聚焦主需求）
        ├─ 长尾词（细分场景延伸，更口语化）
        └─ 询问句（用户在 AI 里实际会输入的完整问句）
```

### 数量阈值

| 词类型 | 每个意图标签的最低数量 |
|-------|---------------------|
| 精准词 | ≥3 个 |
| 长尾词 | ≥5 个 |
| 询问句 | ≥4 个 |

**询问句五类必须覆盖**：**是什么 / 为什么 / 怎么办 / 怎么选 / 哪家好**——缺任何一类都会漏掉一类 AI 引用场景。

### 词库规模指引

| 品牌阶段 | 推荐词库规模 |
|---------|-----------|
| 冷启动（首次建库）| 50-100 词，L3 和 L5 层优先 |
| 系统化运营阶段 | 200-500 词，L1-L5 均衡覆盖 |
| 行业话语权阶段 | 500+ 词，包含大量行业通用词 |

### 词库更新触发条件

以下任一情况触发重审：
- 每季度例行：剔除无效词，补充新流行词
- 平台更新：AI 平台算法或用户行为发生明显变化
- 业务变化：新品上市 / 进入新市场 / 竞品格局调整
- 用户行为迁移：客服/销售反馈出现新的高频问题

## 6. 产出物模板

```
# [品牌名] GEO 意图词库
建立日期：______  最近更新：______  版本：v1.0

#### 词库总览
- 总意图标签数：___  总词量：___
- 品牌类型：ToC（L3+L5 重点）/ ToB（L2+L3 重点）

#### 词库主表

| 层级 | 意图标签 | 精准词 | 长尾词 | 询问句 | 场景类型 | 优先级 | 来源渠道 |
|------|---------|-------|-------|-------|---------|-------|---------|
| L1 | 底妆持妆困扰 | 底妆卡粉 | 为什么底妆上脸半小时就花了 | 底妆卡粉是皮肤原因还是产品原因？ | 认知启蒙型 | 高 | AI下拉/抖音评论 |
| L3 | 持妆粉底液选购 | 干皮持妆粉底推荐 | 干皮用什么粉底液不起皮 | 2026年干皮持妆粉底哪个品牌最值？ | 对比选购型 | 高 | 竞品词库/AI下拉 |
| L5 | 歌剧魅影产品口碑 | 歌剧魅影粉底测评 | 真实用过歌剧魅影粉底的体验 | 歌剧魅影粉底液到底好不好用？ | 品牌评价型 | 高 | 销售反馈/达人内容 |

#### 人群×场景矩阵（深度挖词，选填）

| 目标人群 | 场景 | 生成询问句 | 对应层级 |
|---------|------|---------|---------|
```

**优先级说明**：
- **高**：该层级核心词且与品牌业务强相关，优先安排内容覆盖
- **中**：其他层级词，或与核心业务关联度较弱
- **低**：长尾/偶发性词，有余力时覆盖

## 7. 常见失败模式

### F1：只挖品牌词，忽略品类词和需求词

**症状**：词库里全是"歌剧魅影是什么"/"歌剧魅影哪里买"——全是带品牌名的词，没有品类词和需求词。

**修正**：带品牌名的词不能超过总量的 20%。先建行业词和需求词（L1-L3），品牌词最后补。

### F2：词库建完不更新，变成"死词库"

**症状**：词库建好后再没动过，新趋势词没有补充，旧词没有剔除。

**修正**：每季度例行检查，参照 2.3.5 动态更新触发条件。

### F3：L1-L5 层级分布失衡

**症状**：大量 L1 认知层词，但 L3 评估词和 L5 口碑词极少。

**修正**：ToC 品牌的 L3 + L5 词量之和应占总词库的 40% 以上。

### F4：所有行业都用一个词表

**症状**：把护肤品牌的词套在彩妆品牌上，没有做行业拓展。

**修正**：必须执行五场景骨架的行业填充，"风险规避型"在不同行业含义完全不同。

### F5：只抄竞品词，没挖自身差异化词

**症状**：词库全部参照竞品，最终完全重叠，没有自身卖点对应的词。

**修正**：路径一（自身优势反推）挖出的差异化词是主要来源。先做这一步，再做竞品词参考。

### F6：询问句全是泛问，缺精准决策词

**症状**：全是"歌剧魅影好不好"这类宽泛问题，缺少精准对比词。

**修正**："怎么选"和"哪家好"是最容易触发品牌推荐的句式，占比不低于 40%。

## 8. 执行自检清单

完成意图词挖掘后的验收标准：

- [ ] 品牌属性已明确（ToC/ToB/行业/核心产品）
- [ ] 五场景骨架已填完，每个场景 ≥5 个询问句
- [ ] L1-L5 五层全有词，无明显空缺层级
- [ ] 每个意图标签下精准词 ≥3，长尾词 ≥5，询问句 ≥4
- [ ] 询问句五类（是什么/为什么/怎么办/怎么选/哪家好）均有覆盖
- [ ] 带品牌名词占比 ≤20%
- [ ] 词库已按产出物规范整理成表格
- [ ] 与诊断报告核心缺口清单对照，核心缺口方向均有对应词

## 9. 飞书多维表格同步

词库生成后可同步到飞书多维表格，方便团队协作和持续维护。

### 9.1 配置

编辑 `scripts/config.json`，填入飞书应用凭据和表格信息：

```json
{
  "feishu": {
    "app_id": "cli_xxx",
    "app_secret": "xxx",
    "bitable_app_token": "xxx",
    "table_id": "tblxxx"
  }
}
```

**飞书应用创建步骤**（手动，一次性）：
1. 登录 [飞书开放平台](https://open.feishu.cn/app) → 创建企业自建应用
2. 添加权限：`bitable:app`（多维表格读写）
3. 发布应用，记录 App ID 和 App Secret
4. 创建多维表格，字段：层级 / 意图标签 / 精准词 / 长尾词 / 询问句 / 场景类型 / 优先级 / 来源渠道
5. 将应用添加为表格协作者（或开启链接分享-可编辑）
6. 从表格 URL 中提取 app_token 和 table_id

### 9.2 首次同步（create）

```bash
python scripts/sync_to_feishu.py 边锋-intent-library.md --mode create
```

全量写入所有记录到飞书表格。

### 9.3 增量更新（update）

```bash
python scripts/sync_to_feishu.py 边锋-intent-library.md --mode update
```

增量逻辑（按「意图标签」做 upsert）：
- **已有标签** → 更新该行的精准词/长尾词/询问句等字段
- **新标签** → 新增行
- **旧表有但新生成没有** → 优先级改为「待复审」（不删除，防误删有效词）

### 9.4 何时触发增量更新

参照 2.3.5 词库更新触发条件：
- 每季度例行
- AI 平台算法或用户行为明显变化
- 品牌新品上市 / 进入新市场 / 竞品格局调整
- 客服/销售反馈出现新的高频问题
