## Q2. 双向链接完整性（P1 — 严重级）

**检查方法**：
1. 提取所有知识库页面中的 `[[原始资料/xxx]]` 引用，按资料分组（用 Grep 搜 `\[\[原始资料/`）
2. 对比每篇原始资料的 `## 引用关系` 回链列表：回链中的页面是否确实引用了该资料
3. 计算：有回链的资料数 / 被引用的资料总数

**跨平台命令**：
```bash
# Bash (Git Bash)
grep -r "## 引用关系" D:/Projects/geo-knowledge-base/GEO/GEO理论/原始资料/

# PowerShell
Get-ChildItem -Path "D:/Projects/geo-knowledge-base/GEO/GEO理论/原始资料/" -Filter "*.md" -Recurse | Select-String -Pattern "## 引用关系"
```

**通过标准**：
- [ ] 每篇被知识库引用的原始资料都有 `## 引用关系` 小节
- [ ] 回链列表准确（不引用该资料的页面不出现在回链中）
- [ ] 新 ingest 的资料已生成回链

**自动修复**：无（回链需要人工确认引用关系）

**自动修复**：无（日志和索引需要人工确认内容）

## Q5. CLAUDE.md 进度快照一致性（P1 — 严重级）

**通过标准**：
- [ ] 根层、GEO域层、理论层的进度描述互相不矛盾
- [ ] 理论层的知识地图日期 ≥ 最近操作日期
- [ ] 概念数量、实体数量、SOP 数量与实际文件数一致
- [ ] 本次操作遵守了修改触发矩阵（改什么内容就同步了哪些层级）

**自动修复**：无（涉及跨层同步，需人工判断）

## Q14. 概念/实体/来源摘要同步检查（P1 — 严重级）

**检查方法**：
1. 列出本次 ingest 的原始资料，读取其 frontmatter 中的 `适用概念` 或 `相关概念` 字段
2. 检查这些概念页的最后更新日期是否 ≥ 该资料的 ingest 日期
3. 检查 `来源摘要/` 下是否存在该资料的摘要页
4. 检查相关实体页是否引用了该资料

**通过标准**：
- [ ] 新资料 frontmatter 标注的每个概念，其概念页已更新或确认无需更新
- [ ] 每篇 ingest 的资料都有对应的 `来源摘要/` 页面
- [ ] 实体页的 `已验证行为` 或 `基本信息` 已同步（如资料涉及实体行为）
- [ ] 无"资料 ingest 了但来源摘要缺失"的孤儿

**自动修复**：无（需要人工确认同步关系）

## Q15. AI 操作记录完整性（P2 — 警告级）

**检查方法**：
1. 检查本次结构性改动是否产出了 `AI操作记录/` 下的文件
2. Read 该文件，确认包含：决策路径、关键结论、可引用信息
3. 检查 `日志.md` 最后一条日期是否 = 本次操作日期

**通过标准**：
- [ ] 结构性改动必须有对应的 AI操作记录文件（除非用户明确豁免）
- [ ] AI操作记录包含：背景、问题、决策理由、最终结论
- [ ] 日志.md 已追加，格式正确：`## [YYYY-MM-DD] {类型} | {摘要}`
- [ ] AI操作记录文件的 frontmatter 有 `type: AI操作记录`

**自动修复**：无（需要人工编写决策路径）

## Q17. 引用时效性（P1 — 严重级）

**检查方法**：
1. 抽查知识库页面中的 `来源[Lx]：` 标注，检查对应的原始资料是否存在
2. 对比原始资料的最后修改时间 vs 知识库引用该资料的时间
3. 若原始资料已被更新，检查知识库页面是否仍引用旧版本内容

**通过标准**：
- [ ] 每个 `来源[Lx]：` 标注指向的原始资料文件存在
- [ ] 原始资料更新后，引用它的知识库页面已同步（或确认无需同步）
- [ ] 无"原始资料已删除/重命名但知识库仍引用"的幽灵引用
- [ ] SOP 中的数据/阈值与来源摘要一致

**自动修复**：无（需要人工判断时效性）
